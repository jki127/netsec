Security - Lecture 1

Security Goals

  • Confidentiality is concealment of information
  • Integrity is prevention of unauthorized changes
  • Authenticity is identification and assurance of origin of information
  • Availability is being able to access and use a system

Security Approaches

  • Prevention is stopping an attack
  • Detection is identifying an ongoing or past attack
  • Response is responding to an attack

Physical / Link-Layer Threats

Eavesdropping (sniffing)

  • Network Sniffing is capturing all the communications going through a certain subnet.
  • Using a Network Interface Card's (NIC) promiscuous mode, you can sniff all broadcast communications on the network, not just yours.

Disruption

Overwhelming a WiFi's radio fequency in order to jam the signal is an example of Disruption

Spoofing

  • Spoofing is when an attack creates a message using a bogus source address
  • Usually, an attacker will eavesdrop on the victim's communication and mimic their traffic to match it

Ethernet Basics

MAC address

  • Every Network Interface Card (NIC) has a hard-coded MAC address which is a physical address that is globally unique
  • MAC addresses can be forged using software

Ethernet Frame

  • 1,500 bits is the typical MTU (Maximum Transmission Unit) for Ethernet

CAM Table — Content Addressable Memory

  • CAM tables store MAC addresses and their associated VLAN parameters
  • CAM tables have a limited size
  • CAM table overflow — If a CAM table fills up, everything is broadcasted
  • Overflow can be prevented using Port Security

Port Security Ideas

  1. Allow one hard-coded MAC on each port
  2. Limit how many MAC addresses a port can learn
  3. Set a timer on how long a MAC address will be bound to a port

Internet Protocol — IP

IPv4

  • 4-bytes represent a location on the internet using dotted-decimal notation (ex: 192.168.254.6)

Address Resolution Protocol (ARP)

  • Part of the IP protocol
  • ARP requests are used to get the MAC address associated with a specific IP address
  • ARP requests are broadcasted
  • ARP replies with a node's MAC address are unicast
  • Runs on a single physical network — never transmitted across routers

How ARP works:

  1. I create a request message with:

    • my MAC address
    • my IP address
    • the target IP address
    • a target MAC address field filled with 0

  2. I pass the message to the Data Link Layer to encapsulate it in a frame

    • Source address: my MAC address
    • Destination address: broadcast address

  3. My message is broadcasted

  4. All machines except the targeted one drop the packet

  5. The target message replies only to my machine with an ARP reply containing it's physical address

  6. Reply is received

ARP Table

  • To avoid having to send an ARP request each time, a host can cache the IP and it's associated MAC address in its ARP Table
  • Entries are "aged" meaning that if there is no activity associated with a certain host address for a long period of time, it will be removed from the table

ARP Spoofing

  • ARP is generally implemented as a stateless protocol meaning that systems will usually update their ARP cache if any ARP reply is received even if the system never sent a request for one
  • An attacker can send spoofed ARP replies to convince a system that all of Computer X's frames should go to them — this is know as ARP Poisoning

Man-in-the-Middle Attack

  • Attacker is inserted in between two machines and forwards frames between the two

Force frames to be broadcast

  • Frames can be broadcast by setting the destination to the broadcast MAC (FF:FF:FF:FF:FF:FF)
  • An attacker can spoof ARP replies with the broadcast address which would make it really easy to sniff traffic

Cloning

  • Attacker can assign themselves a computer's IP and MAC to recieve all their intended frames
  • Attacker may first DoS the computer

Defenses against ARP Spoofing

  • Use static ARP entries that can't be updated

    • Spoofed ARP replies are ignored
    • Large overhead

DHCP

  1. Discover
  2. Offer
  3. Request
  4. Ack

DHCP Starvation Attack

  • Attacker tries to lease all the all available DHCP addresses
  • This attack can be mitigated with port security

Layers 3 & 4 - Lecture 3

  • TCP goes over IP
  • TCP is a Layer 4 Protocol
  • IP is a Layer 3 Protocol

Layer 3 Threats

Attacker can:

  • Set a bogus source address
  • Set a bogus destination address
  • Sends tons of packets (flooding)
  • Manipulate routing for eavesdropping

Abrupt Termination via TCP

  • If an attacker knows the ports and sequence numbers, they can disrupt a TCP connection by sending a TCP packet with a RESET (RST) flag

Data Injection via TCP

  • If an attacker knows the ports and sequence numbers, they can send a packet that has malicious data in it

DoS

Reflection Attack

Attacker spoofs source IP address as the intended victim's IP and sends tons of requests. This results in replies all going to the victim's machine.

  • Jan 2001 Attack: Reflection via DNS
